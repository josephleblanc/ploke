## Recommendation

For the first formal Phase 2 packet:

1. Candidate slice:
   `BurntSushi__ripgrep-1294` only.
2. Reference-only supporting artifact:
   `BurntSushi__ripgrep-2209` for replay/introspection and artifact-shape comparison.
3. Binding runtime contract:
   one committed experiment config based on the accepted bounded entry surface from `P2D`.
4. Durable decision record:
   one paired EDR that records why-now, control/treatment, frozen variables, acceptance criteria, validity guards, and explicit waivers.
5. Adopted validity guards:
   `max_provider_failure_rate`, `max_setup_failure_rate`, `require_full_telemetry`, `require_frozen_subset`.
6. Explicit waivers/deferred draft fields:
   `system_prompt_id`, `system_prompt_sha256`, `system_prompt_version`, `tool_schema_version`, `tool_allowlist`, `tool_overrides`, `retry_policy_id`, `timeout_policy_id`, observed `wall_clock_secs`, `tool_implementation_version`, `container_image`, `rust_toolchain`, `seed`, `host_os`, and any `dataset_version` claim beyond the current source descriptor.
7. `record.json.gz` handling:
   optional but preferred replay-grade support when present; not a hard precondition for the first formal packet.
