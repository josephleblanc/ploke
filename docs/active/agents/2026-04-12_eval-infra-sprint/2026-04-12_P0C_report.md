## evidence

  - required syntax/API contract: use `@ 'NOW'` where the present-time query would read validity; `raw_query_at_timestamp(...)` rewrites those markers to `@ <timestamp_micros>` and returns `DbError::QueryConstruction(...)` if none are present
- Current-time vs historical-time proof: [database.rs](</home/brasides/code/ploke/crates/ploke-db/src/database.rs:4886>)
  - `raw_query(...)` at `@ 'NOW'` sees both rows after the second insert
  - `raw_query_at_timestamp(...)` at the saved timestamp sees only the first row
- Explicit marker requirement proof: [database.rs](</home/brasides/code/ploke/crates/ploke-db/src/database.rs:4954>)

  - current query returns one joined row
  - historical query at the earlier timestamp returns zero rows, which requires both validity markers to be rewritten
