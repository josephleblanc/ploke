Scope: `crates/ploke-eval/src/cli/**` CLI-facing reports, monitor records, process boundary outputs, and command structs that read/write Prototype 1 artifacts. Deep state-transition internals (`c1`-`c4`, `child`, `parent`, `journal`, backend/workspace details) were treated as dependencies, not audited directly.

## Highest-Priority Cleanup Items

1. Fix the loop trace path inconsistency: writer uses `<campaign-dir>/prototype1-loop-trace.json`; monitor lists `<campaign-dir>/prototype1/prototype1-loop-trace.json`.
2. Collapse runner result duplication by making `runner-result.json` a latest pointer or derived view over `results/<runtime-id>.json`.
3. Stop embedding full `Prototype1NodeRecord`s in both `scheduler.json` and `nodes/<node-id>/node.json`, or define which copy is authoritative.
4. Replace string status/outcome fields in CLI JSON (`status`, `outcome`, monitor terminal reasons) with typed enums and reserve formatting for table output.
5. Keep branch evaluation canonical in `evaluations/<branch-id>.json`; branch registry should hold only a small decision summary plus artifact path.
