# Workflow Documentation

`docs/workflow` is the durable workflow layer: put templates, schema drafts, framework docs, and skills here when future work should copy or conform to them. Use `docs/active/workflow` for the live programme record that should be updated in place as experiments run.

This directory holds durable workflow assets for eval-driven development:

- templates
- schema drafts
- framework docs
- repo-local skills
- ID conventions
- handoff templates
